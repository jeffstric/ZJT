# 权限装饰器使用说明

## 概述
权限装饰器用于在API接口上进行权限验证，确保只有拥有相应权限的用户才能访问特定接口。

## 装饰器位置
`/home/appuser/comfyui_server_dev/perseids_server/utils/permission.py`

## 可用装饰器

### 1. require_permission - 通用权限验证装饰器

#### 基本用法
```python
from perseids_server.utils.permission import require_permission

@app.post("/api/video-workflow")
@require_permission("video_workflow:create")
async def create_workflow(request: Request):
    """创建视频工作流"""
    pass
```

#### 多权限验证（任意一个）
```python
@app.get("/api/admin/users")
@require_permission(["user:manage_all", "user:admin_switch"], check_mode="any")
async def manage_users(request: Request):
    """管理用户 - 只要有任意一个管理权限即可"""
    pass
```

#### 多权限验证（全部需要）
```python
@app.post("/api/sensitive-operation")
@require_permission(["user:manage_all", "system:admin"], check_mode="all")
async def sensitive_operation(request: Request):
    """敏感操作 - 需要同时拥有所有权限"""
    pass
```

### 2. admin_required - 管理员权限装饰器

#### 基本用法
```python
from perseids_server.utils.permission import admin_required

@app.get("/api/admin/config")
@admin_required
async def get_admin_config(request: Request):
    """获取管理员配置"""
    pass
```

## 参数说明

### require_permission 参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| permission | str 或 List[str] | 是 | 需要的权限代码，如 "video_workflow:view" |
| check_mode | str | 否 | 权限检查模式：<br>- "any": 任意一个权限即可（默认）<br>- "all": 需要所有权限 |

## 权限代码格式

权限代码采用 `模块:操作` 的格式，例如：
- `video_workflow:view` - 查看视频工作流
- `video_workflow:create` - 创建视频工作流
- `user:manage_all` - 管理所有用户
- `computing:view_balance` - 查看算力余额

完整的权限代码列表请参考：`docs/权限系统/function-permission-codes.md`

@require_permission("video_workflow:delete")
async def delete_workflow(request: Request, workflow_id: int):

## 辅助函数

### has_permission - 检查用户权限
```python
from perseids_server.utils.permission import has_permission

# 在业务逻辑中检查权限
if has_permission(user_id, "video_workflow:delete"):
    # 执行删除操作
    pass
else:
    # 返回权限不足错误
    pass
```

### get_user_permissions - 获取用户权限列表
```python
from perseids_server.utils.permission import get_user_permissions

# 获取用户的所有权限
permissions = get_user_permissions(user_id)
# 返回: ["video_workflow:view", "video_workflow:create", ...]
```

### clear_user_permission_cache - 清除权限缓存
```python
from perseids_server.utils.permission import clear_user_permission_cache

# 当用户权限发生变更时，清除缓存
clear_user_permission_cache(user_id)
```

## 错误处理

装饰器会抛出以下HTTP异常：

| 状态码 | 说明 |
|--------|------|
| 401 | 用户未登录或token无效 |
| 403 | 权限不足 |
| 500 | 内部错误 |

## 注意事项

1. **装饰器顺序**：权限装饰器应该放在路由装饰器之后
   ```python
   @app.post("/api/endpoint")  # 路由装饰器在前
   @require_permission("...")   # 权限装饰器在后
   async def handler(request: Request):
       pass
   ```

2. **Request对象**：被装饰的函数必须接收 `Request` 对象作为参数

3. **异步函数**：装饰器支持异步函数，被装饰的函数应该使用 `async def`

4. **权限代码规范**：权限代码必须遵循 `模块:操作` 的格式

5. **缓存更新**：当用户权限发生变更时，必须调用 `clear_user_permission_cache` 清除缓存
