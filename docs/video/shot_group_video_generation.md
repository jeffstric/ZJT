# 分镜组节点视频生成功能

## 功能概述

分镜组节点新增了视频生成功能，可以将整个分镜组的所有分镜合并生成为一个完整的视频。该功能与分镜节点的视频生成功能类似，但具有以下特点：

1. **拼接所有分镜的视频提示词**：汇总整个分镜组中所有子分镜节点的视频提示词，每个镜头标明时间范围
2. **使用第一个分镜的首帧作为图生视频的首帧**：取分镜组中第一个分镜节点的首帧图片
3. **一次性生成视频**：根据抽卡次数生成对应数量的视频（不是为每个分镜单独生成）

## 功能位置

在分镜组节点的节点体中，位于"宫格生图"功能下方，包含以下UI元素：

- **视频模型选择器**：支持 Wan2.2、Sora2、LTX2.0、可灵、Vidu、VEO3.1 六种模型
- **视频时长选择器**：根据选择的模型动态显示可用时长选项
- **生成视频按钮**：带有抽卡次数下拉菜单（X1-X4）
- **算力消耗显示**：实时显示单个视频算力和总算力消耗

## 视频提示词拼接格式

分镜组视频生成时，会将所有子分镜的视频提示词按照以下格式拼接：

```
镜头一：0~4S，[镜头一的视频提示词内容]；镜头二：4~7S，[镜头二的视频提示词内容]；镜头三：7~12S，[镜头三的视频提示词内容]
```

### 时间计算规则

- 镜头一时间：0 ~ 镜头一.videoDuration
- 镜头二时间：镜头一.videoDuration ~ (镜头一.videoDuration + 镜头二.videoDuration)
- 以此类推，累计每个镜头的时长

### 示例

假设有3个分镜节点，时长分别为5秒、5秒、5秒，视频提示词拼接结果为：

```
镜头1：0~5S，夜晚，城市街道，主角走在雨中；镜头2：5~10S，主角进入咖啡厅，坐下点咖啡；镜头3：10~15S，主角看向窗外，表情沉思
```

## 使用流程

### 1. 准备工作

在使用分镜组视频生成功能之前，需要确保：

1. 已创建分镜组节点
2. 已生成分镜节点（通过"生成分镜"按钮）
3. 至少第一个分镜节点已生成首帧图片（通过"生成分镜图"按钮）

### 2. 配置视频参数

1. **选择视频模型**：从下拉菜单中选择合适的视频模型
   - Wan2.2：支持5秒、10秒
   - Sora2：支持10秒、15秒
   - LTX2.0：支持5秒、8秒、10秒
   - 可灵：支持5秒、10秒
   - Vidu：支持5秒、8秒
   - VEO3.1：仅支持8秒

2. **选择视频时长**：根据模型选择可用的时长选项

3. **选择抽卡次数**：点击"生成视频"按钮右侧的下拉箭头，选择X1-X4

4. **查看算力消耗**：系统会实时显示算力消耗（单个视频算力 × 抽卡次数）

### 3. 生成视频

点击"生成视频"按钮，系统会：

1. 检查是否有分镜节点
2. 检查第一个分镜节点是否有首帧图片
3. 拼接所有分镜的视频提示词
4. 调用图生视频API
5. 创建对应数量的视频节点
6. 轮询视频生成状态
7. 更新视频节点的URL

## 算力计算

分镜组视频生成的算力消耗计算公式：

```
总算力 = 单个视频算力 × 抽卡次数
```

不同模型和时长的算力消耗由后端配置决定，前端通过 `getTaskComputingPowerConfig()` 函数获取。

## 错误处理

### 无分镜节点

如果分镜组节点下没有分镜节点，点击"生成视频"按钮时会提示：

```
请先生成分镜节点
```

### 无首帧图片

如果第一个分镜节点没有首帧图片，点击"生成视频"按钮时会提示：

```
第一个分镜节点没有首帧图片，请先生成分镜图
```

### 视频生成失败

如果视频生成失败，系统会显示具体的错误信息，并将按钮状态恢复为"生成视频"。

## 数据保存与恢复

分镜组节点的视频相关数据会自动保存到工作流中，包括：

- `videoModel`：视频模型
- `videoDuration`：视频时长
- `videoDrawCount`：抽卡次数

重新加载工作流时，这些数据会自动恢复。

## 技术实现

### 关键函数

- `generateShotGroupVideo(shotGroupNodeId, shotGroupNode)`：分镜组视频生成主函数
- `calculateVideoComputingPower()`：计算视频生成算力消耗
- `updateVideoComputingPowerDisplay()`：更新算力显示
- `updateVideoDurationOptions(videoModel)`：根据模型更新时长选项

### 数据结构

分镜组节点的 `data` 对象新增字段：

```javascript
{
  videoModel: 'wan22',        // 视频模型
  videoDuration: 5,           // 视频时长（秒）
  videoDrawCount: 1,          // 抽卡次数
  // ... 其他字段
}
```

### API调用

视频生成使用与分镜节点相同的API接口：

```
POST /api/ai-app-run-image
```

参数：
- `image_urls`：第一个分镜节点的首帧图片URL
- `prompt`：拼接后的视频提示词
- `duration_seconds`：视频时长
- `count`：抽卡次数
- `ratio`：视频比例（默认9:16）
- `video_model`：视频模型
- `user_id`：用户ID
- `auth_token`：认证令牌

## 与分镜节点视频生成的区别

| 特性 | 分镜节点 | 分镜组节点 |
|------|---------|-----------|
| 视频提示词 | 单个分镜的视频提示词 | 所有分镜的视频提示词拼接（标明时间范围） |
| 首帧图片 | 当前分镜节点的首帧 | 第一个分镜节点的首帧 |
| 生成数量 | 根据抽卡次数生成多个视频 | 根据抽卡次数生成多个视频 |
| 视频时长 | 用户选择的时长 | 用户选择的时长（不是所有分镜时长之和） |
| 适用场景 | 单个镜头的视频生成 | 整个分镜组的视频生成 |

## 注意事项

1. **分镜组生成的是单一视频**：不是为每个分镜单独生成视频，而是将所有分镜合并为一个视频
2. **首帧取自第一个分镜节点**：需要确保第一个分镜节点有首帧图片
3. **视频时长由用户选择**：不是所有分镜时长的总和，而是用户在下拉菜单中选择的时长
4. **算力计算**：单个视频算力 × 抽卡次数
5. **视频提示词格式**：每个镜头标明时间范围，用分号分隔
