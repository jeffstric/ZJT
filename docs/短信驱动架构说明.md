# 短信驱动架构说明

## 概述

短信发送功能采用了驱动模式（Driver Pattern），支持多种短信发送方式，可以通过配置文件灵活切换。

## 架构设计

### 1. 抽象基类 `BaseSmsDriver`

所有短信驱动都继承自 `BaseSmsDriver` 抽象基类，该类定义了统一的接口：

```python
class BaseSmsDriver(ABC):
    @abstractmethod
    def send_code(self, phone: str, code: str) -> Dict[str, any]:
        """发送验证码短信"""
        pass
```

### 2. 驱动实现

目前支持两种驱动实现：

#### LegacySmsDriver（阿里云SDK驱动）

- **驱动类型**: `legacy`
- **实现方式**: 基于阿里云短信SDK
- **依赖**: `alibabacloud_dysmsapi20170525`
- **适用场景**: 使用阿里云短信服务

#### ApiSmsDriver（HTTP API驱动）

- **驱动类型**: `perseids`
- **实现方式**: 通过HTTP POST/GET请求调用Perseids短信API
- **依赖**: `httpx`
- **适用场景**: 使用Perseids平台的短信服务

### 3. 驱动工厂 `SmsDriverFactory`

工厂类负责根据配置文件中的 `active_driver` 自动创建和管理驱动实例（单例模式）：

```python
# 获取驱动实例（自动根据配置选择）
driver = SmsDriverFactory.get_driver()

# 发送验证码（便捷方法）
result = SmsDriverFactory.send_code(phone, code)
```

## 配置说明

### 配置文件位置

配置文件位于项目根目录：
- 开发环境: `config_dev.yml`
- 生产环境: `config_prod.yml`
- 示例配置: `config.example.yml`

### 配置格式

```yaml
sms:
  # 全局驱动选择（必需）
  active_driver: "legacy"  # 可选值: "legacy", "perseids"
  
  agents:
    # 代理名称（可以配置多个）
    default:
      driver: "legacy"  # 驱动类型
      # ... 驱动特定配置
    
    api_agent:
      driver: "perseids"  # 驱动类型
      # ... 驱动特定配置
```

**说明**：
- `active_driver`：全局驱动选择，系统会自动查找匹配该驱动类型的 agent 配置
- `agents`：各个驱动的具体配置，每个 agent 必须指定 `driver` 字段

### Legacy驱动配置示例

```yaml
sms:
  active_driver: "legacy"  # 使用阿里云SDK驱动
  
  agents:
    default:
      driver: "legacy"
      access_key_id: "LTAI5txxxxxxxx"
      access_key_secret: "eKRGvkHCxxxxxxxx"
      sign_name: "公司名称"
      template_code: "SMS_29225xxxx"
      region_id: "cn-hangzhou"  # 可选，默认cn-hangzhou
```

**必需字段**:
- `access_key_id`: 阿里云AccessKey ID
- `access_key_secret`: 阿里云AccessKey Secret
- `sign_name`: 短信签名
- `template_code`: 短信模板代码

**可选字段**:
- `region_id`: 区域ID，默认 `cn-hangzhou`

### Perseids API驱动配置示例

```yaml
sms:
  active_driver: "perseids"  # 使用Perseids API驱动
  
  agents:
    api_agent:
      driver: "perseids"
      api_url: "http://192.168.10.212:8082/api/v1/live/send_verify_code_simple"
      method: "POST"  # 默认POST
```

**必需字段**:
- `api_url`: 短信API接口地址

**可选字段**:
- `method`: HTTP请求方法，默认 `POST`

### Perseids API驱动请求格式

发送验证码时，Perseids API驱动会构造如下请求：

**POST请求示例**:
```json
{
  "phone": "13800138000",
  "code": "123456"
}
```

### Perseids API驱动响应格式

Perseids API驱动支持以下HTTP状态码：

**成功响应 (200)**:
```json
{
  "success": true,
  "message": "验证码发送成功"
}
```

**错误响应**:
- **400**: 参数错误或手机号格式错误
- **429**: 请求频率限制
- **500**: 服务器内部错误

所有响应均为JSON格式，包含 `success` 和 `message` 字段。

## 使用方法

### 1. 在代码中使用

```python
from perseids_server.utils.sms_drivers import SmsDriverFactory

# 方法1: 使用工厂类的便捷方法（推荐）
result = SmsDriverFactory.send_code(
    phone="13800138000",
    code="123456"
)

# 方法2: 获取驱动实例后使用
driver = SmsDriverFactory.get_driver()
if driver:
    result = driver.send_code("13800138000", "123456")

# 返回结果
# {"success": True, "message": "验证码发送成功"}
# 或
# {"success": False, "message": "错误信息"}
```

### 2. 在VerifyCodeService中使用

`VerifyCodeService.create_verify_code` 方法已自动使用新的驱动架构：

```python
from perseids_server.services.verify_code_service import VerifyCodeService

result = VerifyCodeService.create_verify_code(
    phone="13800138000",
    code_type="register",
    send_sms=True
)
```

**说明**：驱动类型由配置文件中的 `active_driver` 决定，无需在代码中指定。

## 切换驱动

只需修改配置文件中的 `active_driver` 字段即可全局切换驱动：

```yaml
# 使用阿里云SDK
sms:
  active_driver: "legacy"
  agents:
    # ...

# 切换为Perseids API
sms:
  active_driver: "perseids"
  agents:
    # ...
```

**重要**：
- 修改 `active_driver` 后，系统会自动查找匹配该驱动类型的 agent 配置
- 例如：`active_driver: "perseids"` 会自动找到 `driver: "perseids"` 的 agent

## 扩展新驱动

如需添加新的短信驱动，按以下步骤操作：

### 1. 创建驱动类

在 `perseids_server/utils/sms_drivers/` 目录下创建新的驱动文件：

```python
from .base_sms_driver import BaseSmsDriver

class NewSmsDriver(BaseSmsDriver):
    def send_code(self, phone: str, code: str) -> Dict[str, any]:
        # 实现发送逻辑
        pass
```

### 2. 注册驱动

在 `sms_driver_factory.py` 中注册新驱动：

```python
DRIVER_MAP = {
    'legacy': LegacySmsDriver,
    'perseids': ApiSmsDriver,
    'new': NewSmsDriver,  # 添加新驱动
}
```

### 3. 更新 __init__.py

```python
from .new_sms_driver import NewSmsDriver

__all__ = [
    'BaseSmsDriver',
    'LegacySmsDriver',
    'ApiSmsDriver',
    'NewSmsDriver',  # 导出新驱动
    'SmsDriverFactory'
]
```

### 4. 配置使用

```yaml
sms:
  active_driver: "new"  # 使用新驱动
  
  agents:
    custom:
      driver: "new"
      # ... 新驱动的配置
```

## 注意事项

1. **必需配置**: 配置文件中必须设置 `sms.active_driver`，否则驱动无法初始化
2. **配置验证**: 驱动会在初始化时验证配置的完整性
3. **错误处理**: 所有驱动都会返回统一的结果格式 `{"success": bool, "message": str}`
4. **日志记录**: 驱动会自动记录发送成功/失败的日志，包括使用的驱动类型和agent
5. **单例模式**: 工厂类使用单例模式，全局只会创建一个驱动实例
6. **自动匹配**: 系统会自动查找第一个匹配 `active_driver` 的 agent 配置
