# 版本系统常量使用示例

## 导入说明

所有常量类都定义在 `config/constant.py` 中：

```python
from config.constant import Action, Edition
```

## 配置读取

项目统一使用 `config_util.py` 中的函数读取配置文件（带缓存）：

```python
from config.config_util import get_config, get_config_value

# 方式 1：获取整个配置
config = get_config()
db_host = config.get('database', {}).get('host', 'localhost')

# 方式 2：直接获取指定值（推荐）
db_host = get_config_value('database', 'host', default='localhost')
edition_mode = get_config_value('edition', 'mode', default='community')
```

**注意：** 配置文件只在首次调用时读取，后续调用使用缓存，避免重复 I/O 操作。

## Action 类 - 权限操作常量

`Action` 类定义了资源操作的三种权限类型。

### 常量定义

```python
from config.constant import Action

Action.VIEW    # "view"   - 查看权限
Action.EDIT    # "edit"   - 编辑权限
Action.DELETE  # "delete" - 删除权限
```

### 使用示例

#### 1. 在权限检查函数中使用

```python
from config.constant import Action

# 检查查看权限
if _check_resource_permission(world, user_id, Action.VIEW):
    return world_data

# 检查编辑权限
world = _ensure_world_access(world_id, user_id, Action.EDIT)

# 检查删除权限
_ensure_world_access(world_id, user_id, Action.DELETE)
```

#### 2. 在 API 路由中使用

```python
from config.constant import Action

@app.put('/api/worlds/{world_id}')
async def update_world(world_id: int, user_id: int):
    # 编辑操作需要 EDIT 权限
    world = _ensure_world_access(world_id, user_id, Action.EDIT)
    # ... 更新逻辑

@app.delete('/api/worlds/{world_id}')
async def delete_world(world_id: int, user_id: int):
    # 删除操作需要 DELETE 权限
    _ensure_world_access(world_id, user_id, Action.DELETE)
    # ... 删除逻辑

@app.get('/api/worlds/{world_id}/scripts')
async def get_scripts(world_id: int, user_id: int):
    # 查看操作需要 VIEW 权限
    _ensure_world_access(world_id, user_id, Action.VIEW)
    # ... 查询逻辑
```

## Edition 类 - 版本模式管理

`Edition` 类管理系统的版本模式（社区版/商业版）。

### 常量定义

```python
from config.constant import Edition

Edition.COMMUNITY   # "community"   - 社区版
Edition.ENTERPRISE  # "enterprise"  - 商业版
```

### 方法定义

```python
Edition.get_mode()       # 获取当前版本模式
Edition.is_community()   # 判断是否为社区版
Edition.is_enterprise()  # 判断是否为商业版
Edition.get_label()      # 获取版本标签（"社区版"/"商业版"）
```

### 使用示例

#### 1. 判断版本模式

```python
from config.constant import Edition

# 方式 1：使用类方法（推荐）
if Edition.is_community():
    # 社区版逻辑：所有用户共享资源
    query = "SELECT * FROM world"
else:
    # 商业版逻辑：用户资源隔离
    query = "SELECT * FROM world WHERE user_id = %s"

# 方式 2：使用常量比较
if Edition.get_mode() == Edition.COMMUNITY:
    # 社区版逻辑
    pass
```

#### 2. 在 Model 层使用

```python
from config.constant import Edition

class WorldModel:
    @staticmethod
    def list_by_user(user_id: int, page: int = 1, page_size: int = 10):
        where_conditions = []
        params = []
        
        # 商业版才按 user_id 过滤
        if Edition.is_enterprise():
            where_conditions.append("user_id = %s")
            params.append(user_id)
        
        where_clause = " AND ".join(where_conditions) if where_conditions else "1=1"
        # ... 查询逻辑
```

#### 3. 在权限检查中使用

```python
from config.constant import Edition, Action

def _check_resource_permission(resource, user_id: int, action: str) -> bool:
    if Edition.is_community():
        # 社区版：只有删除操作需要检查创建者
        if action == Action.DELETE:
            return getattr(resource, 'user_id', None) == user_id
        return True
    else:
        # 商业版：所有操作都需要检查创建者
        return getattr(resource, 'user_id', None) == user_id
```

#### 4. 在 API 响应中使用

```python
from config.constant import Edition

@app.get('/api/edition')
async def get_edition():
    return {
        'code': 0,
        'data': {
            'mode': Edition.get_mode(),
            'mode_label': Edition.get_label()
        }
    }
```

## GridConfig 类 - 宫格拆分配置

`GridConfig` 类定义了图片宫格拆分相关的配置常量。

### 常量定义

```python
from config.constant import GridConfig

GridConfig.SIZE_2X2              # 4    - 2x2 宫格（标准版）
GridConfig.SIZE_3X3              # 9    - 3x3 宫格（加强版）
GridConfig.VALID_SIZES           # (4, 9) - 允许的宫格大小
GridConfig.DEFAULT_SIZE_BY_TYPE  # {1: 4, 7: 9} - AI工具类型 → 默认宫格大小
GridConfig.LOCK_TIMEOUT_SECONDS  # 120  - 文件锁超时（秒）
GridConfig.IMAGE_DOWNLOAD_TIMEOUT # 60.0 - 下载原图超时（秒）
```

### 使用示例

```python
from config.constant import GridConfig

# 获取默认宫格大小
grid_size = GridConfig.DEFAULT_SIZE_BY_TYPE.get(task_type, GridConfig.SIZE_2X2)

# 验证宫格大小
if grid_size not in GridConfig.VALID_SIZES:
    raise ValueError(f"Invalid grid size: {grid_size}")
```

## 向后兼容性

所有新的类常量都保留了模块级别的别名，现有代码无需修改：

```python
# 旧方式（仍然有效）
from config.constant import TASK_TYPE_GENERATE_VIDEO, AI_TOOL_STATUS_PENDING

# 新方式（推荐）
from config.constant import TaskType, AIToolStatus
task_type = TaskType.GENERATE_VIDEO
status = AIToolStatus.PENDING
```

## 最佳实践

### ✅ 推荐做法

```python
from config.constant import Edition, Action

# 使用类常量，代码更清晰
if Edition.is_community():
    pass

# 使用 Action 常量，避免字符串拼写错误
_ensure_world_access(world_id, user_id, Action.EDIT)
```

### ❌ 不推荐做法

```python
# 避免硬编码字符串
if get_edition_mode() == "community":  # 容易拼写错误
    pass

# 避免使用魔法字符串
_ensure_world_access(world_id, user_id, "edit")  # IDE 无法检查拼写
```

---

## 优势总结

1. **类型安全**：IDE 可以提供自动补全和拼写检查
2. **代码可读性**：`Action.EDIT` 比 `"edit"` 更清晰
3. **易于维护**：修改常量值只需在一处修改
4. **统一管理**：所有版本和权限相关的常量集中在 `edition.py`
5. **向后兼容**：保留函数别名，不影响现有代码
