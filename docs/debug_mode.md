# 视频工作流 Debug 模式使用说明

## 功能概述

Debug 模式是一个开发调试功能，允许开发人员在前端直接查看节点的完整数据结构，方便排查问题和理解节点状态。

## 启用 Debug 模式

### 1. 配置密码

在配置文件中设置 Debug 模式密码：

```yaml
# config.yml
frontend:
  debug_password: "your_debug_password"  # 设置你的 Debug 密码
```

默认密码为 `debug123`（在 `config.example.yml` 中定义）。

### 2. 开启 Debug 模式

1. 打开视频工作流页面
2. 在页面顶部找到 **Debug** 按钮（位于"重置"按钮旁边）
3. 点击 **Debug** 按钮
4. 在弹出的密码输入框中输入配置的密码
5. 密码验证通过后，Debug 模式将被激活

### 3. 视觉反馈

- Debug 按钮会变为绿色，表示 Debug 模式已开启
- 所有节点的标题栏会出现一个 🐛 调试按钮

## 使用 Debug 功能

### 查看节点信息

1. 确保 Debug 模式已开启
2. 找到需要调试的节点
3. 点击节点标题栏中的 🐛 按钮
4. 节点的完整信息将输出到浏览器控制台

## 节点编辑区折叠交互

### 选择性折叠机制

节点内容分为两类：
- **核心信息**（`.field-always-visible`）：始终显示，包括预览图、提示词文本、状态信息等
- **编辑控件**（`.field-collapsible`）：仅在节点选中时显示，包括按钮、下拉选择、文件上传等

### 各节点显示规则

| 节点类型 | 始终显示 | 选中时显示 |
|---------|---------|-----------|
| **视频节点** | 视频预览、状态 | 上传控件、操作按钮 |
| **图片节点** | 图片预览 | 上传、编辑提示词、模型选择、编辑按钮 |
| **图生视频节点** | 首帧/尾帧预览、提示词文本 | 上传、时长/模型选择、生成按钮 |
| **剧本节点** | 剧本信息、字符计数 | 输入框、上传、拆分按钮 |
| **分镜组节点** | 分镜列表 | 模型选择、查看/编辑按钮、生成按钮 |
| **分镜节点** | 分镜信息、提示词文本、图片/首帧预览 | 编辑按钮、生成按钮、模型/时长选择 |
| **角色节点** | 参考图、角色属性 | 编辑按钮、生成视频/角色卡按钮 |
| **对话组节点** | 对话列表、音频播放器 | 情感控制、生成按钮 |

### 交互行为

1. 未选中节点默认折叠编辑控件，仅显示核心信息。
2. 点击节点后，节点进入选中状态并展开编辑控件。
3. 点击画布空白区域会取消选中并收起编辑控件。
4. 多选节点时，所有被选中的节点都会展开编辑控件。

### 输出内容

点击调试按钮后，控制台会输出以下信息：

- **ID**: 节点的唯一标识符
- **Type**: 节点类型（如 `video`, `image`, `script`, `shot_frame` 等）
- **Title**: 节点标题
- **Position**: 节点在画布上的坐标 `{x, y}`
- **Data**: 节点的完整数据对象（包含所有配置、状态、文件引用等）
- **完整节点对象**: 节点的完整对象引用

### 查看控制台

在浏览器中打开开发者工具：

- **Chrome/Edge**: 按 `F12` 或 `Ctrl+Shift+I` (Windows/Linux) / `Cmd+Option+I` (Mac)
- **Firefox**: 按 `F12` 或 `Ctrl+Shift+K` (Windows/Linux) / `Cmd+Option+K` (Mac)
- **Safari**: 按 `Cmd+Option+C` (Mac)

切换到 **Console** 标签页查看输出的节点信息。

## 关闭 Debug 模式

1. 再次点击页面顶部的 **Debug** 按钮
2. Debug 按钮会恢复原始样式
3. 所有节点的调试按钮会自动隐藏

## 注意事项

1. **仅用于开发调试**: Debug 模式仅供开发人员使用，不应在生产环境中启用
2. **密码保护**: 请妥善保管 Debug 密码，避免泄露
3. **性能影响**: Debug 模式本身不会影响性能，但频繁输出大量节点数据可能会占用控制台资源
4. **浏览器兼容性**: 确保使用现代浏览器（Chrome、Firefox、Edge、Safari 最新版本）

## 常见使用场景

### 1. 排查节点数据问题

当节点行为异常时，可以通过 Debug 模式查看节点的完整数据，检查：
- 数据是否正确保存
- 文件 URL 是否有效
- 配置参数是否符合预期

### 2. 理解节点结构

开发新功能时，可以查看现有节点的数据结构，了解：
- 节点数据的组织方式
- 不同类型节点的数据差异
- 连接关系的存储方式

### 3. 验证工作流保存/加载

检查工作流保存和加载功能是否正常：
- 保存前查看节点数据
- 重新加载后再次查看
- 对比数据是否一致

## 技术实现

### 前端实现

- **状态管理**: 在 `state.js` 中添加 `debugMode` 状态
- **UI 控制**: 在 `video_workflow.html` 中添加 Debug 切换按钮
- **事件处理**: 在 `workflow.js` 中实现密码验证和 UI 更新
- **节点按钮**: 在各节点创建函数中调用 `addDebugButtonToNode()` 添加调试按钮

### 后端实现

- **配置管理**: 在 `config.yml` 中添加 `frontend.debug_password` 配置项
- **API 接口**: 在 `server.py` 中添加 `/api/config/debug-password` 接口返回密码

### 安全考虑

- 密码验证在前端进行，主要用于防止误操作
- 不应将 Debug 密码视为高安全性的保护措施
- 生产环境建议禁用此功能或使用强密码
