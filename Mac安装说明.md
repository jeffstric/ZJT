# ZJT Server - Mac 安装说明

---

##  小白版本 - 纯图形化操作(推荐新手)

### 首次安装(只需操作一次)

#### 步骤 1️⃣: 尝试打开文件
在文件夹中**双击** `首次安装-Mac.command` 文件,会弹出安全警告

#### 步骤 2️⃣: 通过系统设置允许
1. 点击对话框的 **"Done"** 按钮
2. 点击屏幕**左上角的苹果图标** 🍎 → 选择 **"系统设置"**
3. 在左侧菜单找到 **"隐私与安全性"**
4. **向下滚动**到页面底部
5. 找到 "首次安装-Mac.command 已被阻止" 的提示
6. 点击旁边的 **"仍要打开"** 按钮
7. 输入 Mac 的**登录密码**确认

#### 步骤 3️⃣: 再次运行
回到文件夹,再次**双击** `首次安装-Mac.command`
- 会自动打开终端窗口
- 等待显示 "安装完成!"
- 按回车键关闭窗口

#### 步骤 4️⃣: 完成!
现在桌面上会出现 `ZJT Server.app` 应用程序

### 日常使用
**双击 `ZJT Server.app`** 即可启动服务

> 💡 首次运行 `ZJT Server.app` 时,也需要通过系统设置允许(重复步骤 2),之后就可以直接双击了

### 如何停止服务
在终端窗口中按键盘上的 **Control + C** 键

## 💻 大牛版本 - 命令行快速执行

### 一键修复权限（首次运行）
打开终端，进入项目目录，执行：

```bash
xattr -cr .
```

这会移除所有文件的隔离属性，之后就可以正常运行了。

### 一键安装
打开终端,进入项目目录,执行:

```bash
bash scripts/tools/create_mac_app.sh
```

### 日常启动
```bash
./start.command
```

或者 双击 `ZJT Server.app`** 即可启动服务


### 停止服务
在终端窗口中按 **Control + C**

## ❓ 常见问题

### Q: 小白版本 - 找不到"隐私与安全性"?

1. 打开 **系统设置**
2. 在左侧边栏向下滚动
3. 找到 **"隐私与安全性"** 图标并点击

### Q: 小白版本 - 系统设置中没有"仍要打开"按钮?

必须先**尝试打开文件**(双击或右键打开),触发安全警告后,系统设置中才会出现"仍要打开"选项

### Q: 如何查看项目路径?

在 Finder 中打开项目文件夹,按 **Command + Shift + C** 复制路径,或右键点击文件夹标题栏选择"拷贝...为路径名称"

## 📁 项目结构说明

为了更好的项目组织，部分脚本已重新分类到 `scripts/` 目录下：
- `scripts/tools/` - 工具脚本（如 create_mac_app.sh）
- `scripts/launchers/` - 启动器脚本
- `scripts/running/` - 运行脚本

**注意**: 这不影响日常使用，所有启动方式保持不变。
