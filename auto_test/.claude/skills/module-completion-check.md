# 模块完成度验证技能

## 目的

验证单个模块的测试是否真正完成，防止部分测试就草率结束的问题。

## 🔧 核心工具：test_navigator.py

**使用 Python 脚本检查模块完成状态，避免直接读取 JSON 文件！**

```bash
# 检查指定模块的详细状态
python test_navigator.py --module <模块ID>

# 查看所有模块的完成情况
python test_navigator.py --list

# 查看整体进度统计
python test_navigator.py --status
```

## ⚠️ 模块完成标准

**单个模块必须满足以下条件才算测试完成：**

1. **步骤完成率：100%** - 所有测试步骤都必须通过
2. **功能完成率：100%** - 所有功能都必须完成
3. **无待测试项** - `python test_navigator.py --module <模块ID>` 应显示 "[DONE] 模块的所有测试已完成！"

## 验证流程

### 1. 检查模块状态

```bash
python test_navigator.py --list
```

**不合格示例：**
```
4. [RUNNING] [node_operations] 节点操作模块
   功能进度: 8/21 (8/21)
   步骤进度: 28/88        # ❌ 只有 31.8%，未完成

7. [RUNNING] [timeline] 视频时间轴模块
   功能进度: 3/15 (3/15)
   步骤进度: 14/76        # ❌ 只有 18.4%，未完成
```

**合格示例：**
```
4. [DONE] [node_operations] 节点操作模块
   功能进度: 21/21 (21/21)
   步骤进度: 88/88        # ✅ 100% 完成

7. [DONE] [timeline] 视频时间轴模块
   功能进度: 15/15 (15/15)
   步骤进度: 76/76        # ✅ 100% 完成
```

### 2. 验证无待测试项

```bash
python test_navigator.py --module node_operations
```

**应该显示：**
```
[DONE] 模块 [node_operations] 的所有测试已完成！
```

**如果显示具体测试项，说明还有未完成的测试。**

### 3. 检查整体进度

```bash
python test_navigator.py --status
```

确认该模块在整体统计中显示为完成状态。

## 常见问题和解决方案

### 问题1：步骤未达到100%

**现象：** 步骤进度显示如 "28/88" 或 "14/76"

**解决方案：**
1. 运行 `python test_navigator.py --module <模块ID>` 找到下一个待测试项
2. 继续执行剩余的测试步骤
3. 使用 `python test_navigator.py --pass-current --module <模块ID>` 标记完成

### 问题2：功能未全部完成

**现象：** 功能进度显示不是 100%

**解决方案：**
1. 检查是否有功能的某些步骤未通过
2. 重新执行失败的测试步骤
3. 确保每个功能的所有步骤都标记为通过

### 问题3：模块状态仍显示 [RUNNING]

**现象：** 模块列表中状态不是 [DONE]

**解决方案：**
1. 检查是否所有步骤都已标记为通过
2. 验证会话文件是否正确更新
3. 重新运行 `python test_navigator.py --list` 确认状态

## 验证清单

在声明模块测试完成前，必须检查：

- [ ] 运行 `python test_navigator.py --list`，确认模块状态为 `[DONE]`
- [ ] 步骤进度显示为 `X/X`（分子分母相等）
- [ ] 功能进度显示为 `X/X`（分子分母相等）
- [ ] 运行 `python test_navigator.py --module <模块ID>` 显示 "[DONE] 所有测试已完成"
- [ ] 整体进度统计中该模块计入已完成模块数

## 报告格式

模块完成后，使用以下格式报告：

```
✅ [模块名] 测试完成
   - 功能完成：21/21 (100%)
   - 步骤完成：88/88 (100%)
   - 验证状态：[DONE]
   - 无待测试项：已确认
   - 备注记录：已为关键步骤添加测试备注
```

**备注字段说明：**
- 测试清单中每个步骤都有 `remark` 字段
- 测试智能体可以在标记步骤通过时添加备注
- 备注信息保存在会话文件中，用于测试报告生成
- 项目工程师可以通过备注了解测试过程中的具体发现

## ❌ 严格禁止的行为

**绝对禁止以下行为：**

1. **禁止提前标记完成** - 不允许在测试未完成时就标记模块为完成
2. **禁止跳过复杂测试** - 不允许因为测试复杂就跳过或延后
3. **禁止假设完成** - 不允许假设剩余测试会通过就提前标记

### 错误示例（严格禁止）

```
❌ 错误：这是一个复杂的测试，有11个步骤。但首先让我检查节点模块是否还有其他更简单的测试。
        让我先标记节点模块测试为完成，然后检查时间轴模块。

❌ 错误：大部分功能都测试通过了，我先标记这个模块为完成。

❌ 错误：剩下的测试看起来都很简单，应该会通过，我先标记完成。
```

### 正确做法

```
✅ 正确：继续执行当前测试步骤，无论多复杂都必须完成。
✅ 正确：只有在所有步骤都实际执行并通过后才标记完成。
✅ 正确：使用脚本验证 100% 完成后再声明模块完成。
```

## 重要提醒

**绝对不要在步骤未达到100%时声明模块测试完成！**

即使大部分功能已测试，只要还有一个步骤未完成，整个模块就不算完成。这是为了确保测试的完整性和可靠性。

**必须先完成所有测试，再标记完成，而不是先标记完成再去测试！**
